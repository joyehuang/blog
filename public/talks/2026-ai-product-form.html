01 / 缘起

那场面试,来自一家"文档协作 → AI 版 Canva"的公司

创始人做石墨文档这类文档协作 SaaS 起家,现在转头做 AI native 的设计工具。我做好了被拷打一小时的准备——结果整场没在考"我背没背过",而是对着他们产品里那些还没想清楚的问题,聊了一个多小时。
聊完我脑子里的问题不是"我答得怎么样",而是:AI 时代,一个软件产品到底该长成什么样?
02 / 背景 · 这条赛道有多大

设计 / 创作工具,是个正在被 AI 重写的巨型市场

Canva
~$42B
  • 260M 月活 · 29M 付费
  • 年化收入 $3.5B+,连续 8 年盈利
  • $370M 收购 Leonardo AI 自建模型
Figma
~$10B
  • 2025 上市,一度冲到 ~$59B
  • 到 2026 年已腰斩
  • 市场在为"AI 颠覆设计工具"定价
石墨 / 这类
转型中
  • 国内第一款实时协作云文档
  • 设计驱动 · 五轮融资
  • 正从文档协作 → AI 设计
读数 › 一家上市设计公司因"AI 会不会颠覆我"被腰斩——这本身就是今天命题的入口。
03 / 产品论① · 产品形态的分野

Figma 很强,但它要"设计功底";Canva 不要

Figma 强在专业、上限高,但门槛也高;Canva 中性化、人人可上手。在 AI 时代,产品该往哪一侧站,其实很清楚。
Figma
专业 · 高门槛
Canva
大众 · 零门槛
AI native 的设计工具
推论 › 当所有人都能做出干净的设计,优势就从"会不会设计"转移到"差异化、产品选择、执行速度"。
04 / 产品论② · 交互范式的跃迁

从"操作工具",到"表达意图"

Figma 要你会操作工具;AI native 的工具,你只要表达意图——专业那部分,交给模型补。门槛塌掉的本质,是交互范式变了。
注意:这跟我面试被问的"意图识别"是同一件事的两面——产品侧叫"表达意图",技术侧叫"意图识别"。产品愿景和技术难点,是同一枚硬币。
05 / 放大命题

别只盯着"Agent",
盯"AI 时代软件该怎么长"

我们不该被"Agent"这个词框住。真正的问题是:AI 时代,软件本身的形态在变。
趋势 › Gartner:到 2028,约 1/3 的用户体验会从"原生 App"转向 agentic 前端。SaaS 正从"数据库 + 表单",变成"会干活的 worker"。
06 / 核心论点

我的判断:通用 Agent 走不通,垂类才有护城河

号称首个通用 agent 的 Manus,2025 底被 Meta 收编、暂停注册;横向通用市场仍在混战,但垂类已经分化出胜负——护城河在"领域上下文 > 通用推理"的地方。
垂类 SaaS · 年增18–32%
横向通用 · 年增12–15%
现实 › 若把 Claude Code、Cursor 当通用 agent(强在 coding、越来越通用),普通开发者正面 PK 打不赢——只能做垂类、深耕行业 know-how。Atypica(商业研究)就是垂类样本。
07 / 反直觉的一面

垂类做完了,你还得"接"通用能力

为什么?技术变化太快。哪怕产品形态做完、需求写死,后面还是要把 Claude Code、Codex 这类通用能力,通过插件 / MCP 接进来。
证据 › 多数 2025–26 领头羊把垂类 agent 叠在现有 SaaS 上(agent-over-SaaS),MCP 已成标准集成层,OpenAI、Google、微软都接了;Canva 现在也能被 Claude Code / Codex 通过 MCP 双向调用。
08 / 价值的迁移

价值,正从"单个应用"挪到"编排层"

从"集成很多 SaaS 工具",到"一个 agent 编排所有系统"。价值从任何单一应用,转移到连接它们的编排层——就像当年从单体架构走向微服务。
现在的 agent 生态 ≈ 2008 年的云:AWS 还没铺满、Stripe 还没出现。每一层都是机会——包括底下"卖铲子"的基础设施。
09 / 回到那场面试

我被问的题,没一道在问"模型行不行"

给 agent 灌审美(taste)
→
用户偏好的长期记忆
用户丢参考图 / PDF / CSV / PPT,意图含糊
→
意图识别 + 文件归类
截图预览要两个 tool call
→
成本怎么压
除了 O 系列和 GPT,哪个模型更胜任
→
模型按任务路由
全是"模型外面那层(垂类 + 集成)的工程"。题目本身,就是趋势数据。
10 / 那这跟"我"有什么关系

AI 让知识廉价,面试称重的是经验

"意图识别""文件系统"这种 System Design 大题,常人要 1 分钟构思,我 5–10 秒就讲完——不是反应快,是我真做过。经验,是 AI 顶不掉、面试也装不出来的东西。
这也是我能拿到对标正式员工薪资的原因:在没有标准答案的地方,你能做对判断。薪资,是判断力的定价。
11 / 给开发者的四条

如果你也在做产品 / 找方向

做产品
往"表达意图"那侧走
别要求用户有专业功底,让模型补专业。
选方向
做垂类,深耕行业 know-how
护城河 = 领域上下文 + 系统集成 + 结果定价。
搭架构
留好 MCP / 插件口
技术变化快,准备好随时接通用能力。
做自己
攒经验和判断力
AI 顶不掉、面试装不出,这才是溢价来源。
